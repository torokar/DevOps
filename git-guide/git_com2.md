### Управления ветками

**git branch** - Команда используется для управления ветками в репозиторит Git. 
с ее помощью можно создавать, удалять и перечислять ветки. Ветки в Git позволяют 
вам изолировать работу на различных функциях, исправлениях или версиях проекта

**git branch <названия ветки>** - Создания ветки с вольным названием 

## Основные команды 

1) git branch -d <названия ветки> - удаляет указанною ветку

2) git branch -D <названия ветки> - принудительно удаляет ветку, даже если она не была объединена

3) git branch --move <старое названия> <новое названия> - изменить названия ветки 
или коротко git branch -m <старое названия> <новое названия>

4) git branch -a(or --all) - показывает все ветки: локальные и удаленные

5) git branch -r(or --remotes) - показывает только удаленные ветки

6) git branch <name> <commit> - создает ветку на укащанном коммите или теге

### Переключение между ветками

**git switch** - Команда используется для переключения между ветками

``` git switch <названия ветки> ``` 

Мы также могли создать ветку при помощи **git switch** и сразу переключится на нее при помощи флага -с

**git switch -c <названия новой ветки>**

Пример:

```
Добавляем новый файл 
touch index.html 

git add . 

git commit -m "first commit"

git push
```

```

git push -u origin <названия ветки> 
Отправка новой ветки в удаленный репозиторий

```

```
git switch -c(--create) <имя> - cоздает новую ветку и переключается на нее 

git switch -t(--track) - устанавливает связь с отслеживаемой удаленной веткой
```

## Временное хранилище 

```
git stash
```

Команда используется для временного сохранения изменений в рабочем каталоге, которые еще не готовы для коммита. Это полезно, когда нужно переключится на другую ветку или работу, но текущие изменения не должны быть потеряны. С помощью **git stash** изменения созраняются в специальное место, откуда их можно восстановить позже.

*Коротко вообще если мы сломали код и при срочной задаче, чтобы не пушить изменения и не коммитить их. Ложим их так сказать в сторону, а после уже можем вернутся к нему и продолжить работу*

### Основные команды **stash**

```
1) git stash - оправка изменений в стэш
2) git stash push -m "Временные изменения" - Отправка изменений в стэш с указанием сообщения
3) git stash list - список сохраненных в стэш
4) git stash apply - применение последнего стэш
5) git stash pop - применение и удаление последнего стэш 
6) git stash apply stash@{n} - применение конкретного стэш
7) git stash drop stash@{n} - если 0, то удаление всех стэшей, другое число - удаление определенного стэш 
8) git stash clear - очистка всех стэшей
```
