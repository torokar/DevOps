# Процессы 

## Типы процессов

- Демон - это тип процесса в **Linux**, который работает в фоновом режиме без
непосредственного взаимодействия с пользователем. В отличие от интерактивных
эти процессы отделены от конкретной пользовательской сессии и продолжают 
работать, даже если пользователь не вошел в систему. Демоны предназначены для
длительного выполнения и часто запускаются при возникновении определенных событий
в ОС


- Интерактивный процесс работает на переднем плане и взаимодействует напрямую с
пользователем через терминал или другой интерфейс. Такой процесс получает ввод 
от пользователя и отображает результаты в режиме реального времени. Во время
выполнения процесса переднего плана терминал недоступен для других команд
или задач


- Родительский процесс - это процесс, который создает другой процесс, известный
как дочерний. Соответственно, **дочерний процесс** - это тот, который создается 
другим процессом(родительским)

## Просмотреть родительское процессы можно с помощью команды 

```
pstree -p
```

Отображает все запущенные родительские процессы вместе с соответствующими 
дочерними процессами и соответствующими PID

Чтобы сократить вывод исщем по ключевому слову

```
pstree -p | grep 'firefox'
```


# Создание процессов в Linux

Существуют несколько методов **создание процессов**

+ **fork()** - используется для создания нового процесса путем дублирования
существующего. Новый процесс является точной копией родительского 

+ **exec()** - используется для замены текущего образа процесса на новый. 
Этот метод позволяет выполнить другую программу в текущем процессе

+ **system()** - создает новый процесс для работы команды, указанной в качестве
аргумента. Метод ожидает завершения выполнения команды и возвращает статус ее
выхода

+ **clone()** - похож на **fork()**, но обеспечивает больший контроль над 
поведением нового процесса. Он позволяет указывать различные флаги
для управления совместным использованием ресурсов между родительским и дочерним 
процессами


# Определение идентификатора процесса

Для определения **PID** существует утилита **pidof**. Чтобы она определила 
идентификатор процесса, нужно просто указать его название в качестве аргумента

```
pidof <аргумент>

pidof firefox
```

Можем определить идентификатор процесса текущей оболочки **bash**

```
echo $$
```

Чтобы узнать идентификатор родительского процесса этой оболочки, выполните команду

```
echo $PPID
```

# Запуск процесса в Linux

Начнем с **интерактивных процессов** Как было сказано ранее, чтобы их запустить
нужно выполнить команду в окне терминала. Например **wget** для загрузки 
файла из Интернета 

```
wget https://wordpress.org/latest/zip
```

**Фоновый процессы** обычно создаются системой или какой-либо программой. Однако
Мы тоже можем их создать. Для этого в конце команды терминала нужно поставить 
пробел и добавить сивол & (амперсант) И чтобы проверить, что процесс действительно
запущен и работает в фоновом режиме ввести команду **jobs**

```
wget https://wordpress.org/lates.zip & jobs
```

Вы также можете **останавливать текущие активные процессы**. В большинстве случаев
это можно сделать, нажав комбинацию клавиш **Ctrl + c или Ctrl + c** (ctrl + c 
прерывает работу процесса а ctrl + z останавливает)

Далее можно возобновить процесс в фоновом режиме используя команду **bg**

```
bg
``` 

и чтобы снова вывести процесс на передний план выполним команду fg вместе с 
идентификатором **%1**

```
fg %1
```


# Состояния процесса в Linux 

Все это время, пока мы запускали и останавливали процесс, он менял свое состояния
Всего существует пять возможных статусов процесса:

+ Running (R) - непосредственно запущен и активно использует ресурсы процессора

+ Uninterruptible Sleep (D) - Непрерывно спящее состояние, во время которого 
процесс ожидает определенного события; такой процесс не реагирует ни на какие 
сигналы

+ Interruptible Sleep (S) - спящее состояние, во время которого процесс
может реагировать на посылаемые ему сигналы 

+ Stoppe (T) - Состояние, когда процесс был остановлен либо пользователем,
либо сигналом

+ Zombie (Z) - завершенный процесс, который закончил свое выполнение, но все
еще имеет запись в таблице процессов


# Просмотр активных процессов 

В операционной системе Linux существует несколько встроенных инструментов для 
просмотра процессов. Самый известные из них - *ps* *top*

Чтобы утилита ps вывела на экран все запущенные процессы, введите ее в терминале
вместе с опцией(флагом) **-А**

```
ps -A
```

Утилита **top** представляет собой инструмент мониторинга, работающий в реальном
времени. Для ее использования введите 

```
top
```


# Отправка сигналов процессам

Для управления процессами используется сигналы которые отправляются с помощью
команды **kill**, **pkill**, **killall**. Чтобы узнать все доступные сигналы, 
введите следующую команду


**kill** - убивает один процесс по PID 

**pkill** - убивает процессы по имени\шаблону ищет по названию

**killall** - убивает процессы по точному именни убивает все процессы с таким 
именем

```
kill -l
```

Примеры:

1) kill

```
kill 1234  # SIGTERM процессу с PID 1234

kill -9 1234   # SIGKILL

kill -15 1234  # SIGTERM явно

#Типичный сценарий

ps aux | grep nginx #nginx это названия службы или программы

kill -TERM 5678 # Вежливое завершение (можно без -TERM)

sleep 5 

kill -KILL 5678  #Если не помогло - принудительно
```

Где найти PID
```
pgrep nginx # получить PID по имени

pidof nginx # альтернатива

ps aux | grep nginx # Ручной поиск

cat /var/run/nginx.pid  # из pid-файла 
```

2) pkill

```
#Простейшее использование

pkill nginx  #Убить все процессы nginx

pkill -9 nginx   #принудительно

#Поиск по части имени (regex)

pkill "^python"  #все процессы, начинающиеся с python

pkill ".server$"  #все процессы заканчивающиеся на server

# Фильтрация по другим критериям 

pkill -u www-data  # все процессы пользователя www-data

pkill -t pts/0  # все процесс в терминале pts/0

pkill -f "server.ру"  # по полной командной строке (а не только по имени)

# Базовый режим

pkill -l nginx  # только показать, что будет убито

pkill -i nginx  # игнорировать регистр
```

3) killall 

```
#Базовое использование 

killall nginx  # Убить все nginx

killall -9 nginx  # Принудительно

# Полезные опции 

killall -i nginx  #спрашивать подтверждение для каждого

killall -v nginx  # подробный вывод (что убить)

killall -w nginx   # Ждать завершения процессов

# По времени и другим критериям

killall -o 1h nginx   # Процесс, живущие больше часа

killall -y 30m nginx  #Процессы живущие меньше 30 минут

killall -u www-data  #все процессы пользователя

```

Как мы можем заметить у каждого сигнала есть свой номер. По умолчанию утилиты 
**kill**, **pkill**, и **killall** отправляют сигнал **SIGTERM (15)**
предназначенный для **завершения процесса**

Для примера, узнаем **PID** процесса *firefox* с помощью **pgrep** и завершим его,
используя **kill**

```
pgrep -u user firefox

kill 4436
```

Чтобы отправить процессу другой сигнал, нужно указать его номер или название 
вместе с дефисом (-) в качестве аргумента к командам **kill, pkill и killall**.
Для примера отправим процессу *firefox* сигнал SIGKILL (9) 

```
pgrep -u user firefox

kill -9 6504
```


```
pgrep -u user firefox 

kill -SIGKILL 6504
```

Также процесс можно завершить по его названию. Для этого используется комада 
**pkill и killall**

```
pkill firefox

killall firefox
```

# Изменение приоритета процесса Linux

Приоритет процесса влияет на количество получаемого им процессорного времени 
и порядок выполнения. Значение приоритета может быть 
от 19(минимальное) до -20(максимальное). Уменьшать приоритет можно с правами 
обчного пользователя, но чтобы его уеличить нужны права суперпользователя. 
Утилита **top** которую мы применяли ранее, позволяет пользователю узнать 
приоритет каждого процесса - для этого обратите внимание на значения в колонке
**NI**

Для установки приоритета используется команда **nice**

```
nice -n <priority_value> <process>
```

Чтобы изменить приоритет, выполните команду **renice**

```
renice <priority_value> -p <PID>
```
